## Repository layout

Three independently buildable services orchestrated via Docker Compose:

- `backend/` â€” Java 21 + Micronaut 4.7 REST API (`com.cloudsync` package). Built with Gradle Kotlin DSL, runs on JVM in dev and as a GraalVM native image in prod. Persists to SQLite via Micronaut Data JDBC + Flyway 9 (Flyway 10 dropped SQLite support, do not upgrade).
- `icloud-service/` â€” Python 3.12 FastAPI microservice wrapping `icloudpy`. Internal-only (not port-exposed in prod), proxied by the backend. Source under `app/`, tests under `tests/`.
- `frontend/` â€” Angular 21 SPA using standalone components and the new application builder (esbuild + Vite). Tests run under Vitest (not Karma). API client is generated by Orval from `openapi.yml` â€” `prestart`/`prebuild` hooks regenerate it automatically.

## Common commands

### Whole stack (Docker)
```bash
docker compose -f compose.dev.yaml up --build   # dev: hot-reload all 3 services
docker compose -f compose.yaml up -d            # prod-like
```
Dev URLs: frontend `:4200`, backend `:8080`, icloud-service `:8000`. Default credentials `admin` / `changeme`.

### Backend (`backend/`)
```bash
./gradlew run                                   # JVM mode
MICRONAUT_ENVIRONMENTS=dev ./gradlew run        # uses ./dev-cloud-sync.db
./gradlew test                                  # all tests
./gradlew test --tests "com.cloudsync.SomeTest" # single test
./gradlew nativeCompile                         # GraalVM native (~5 min)
./gradlew exportSwagger                         # regenerate ../openapi.yml from annotations
```

### Frontend (`frontend/`)
```bash
npm start              # ng serve on :4200 (auto runs `generate-api` first)
npm run build          # production build (auto runs `generate-api` first)
npm test               # vitest
npm run generate-api   # regenerate Orval client from ../openapi.yml
```

### iCloud service (`icloud-service/`)
```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
pytest tests/
pytest tests/test_foo.py::test_bar              # single test
```

## Architecture notes

**API contract is OpenAPI-driven.** The backend generates `openapi.yml` at the repo root (via Micronaut OpenAPI annotation processor + the `exportSwagger` Gradle task), and the frontend's HTTP client is generated from it by Orval. After changing a backend controller/DTO that affects the API surface: regenerate `openapi.yml`, then regenerate the frontend client (or just `npm start`/`build` which does it via the pre-script).

**Backend package structure** (`com.cloudsync.*`): `controller` (HTTP endpoints) â†’ `service` (business logic) â†’ `repository` (Micronaut Data JDBC) â†’ `model.entity` / `model.dto` / `model.enums`. Cross-cutting: `config`, `exception`, `client` (HTTP clients to icloud-service), `util`.

**LLM agent subsystem** (`com.cloudsync.agent`): A pluggable LLM provider chain used by `DiskDetectionAgent` (and exposed via `AgentController` with SSE streaming). `LlmProvider` has two implementations: `OpenRouterLlmProvider` (real, activated when `OPENROUTER_API_KEY` is non-empty; default model `google/gemini-2.0-flash-001`) and `FallbackLlmProvider` (heuristic, always present as `@Singleton`). `application.yml` sets `openrouter.api-key: ${OPENROUTER_API_KEY:}` so an unset/empty env var transparently falls through to the heuristic. Tools the agent can call live in `agent/tools/`.

**External drive bind-mount with `rshared` propagation.** `compose.dev.yaml` bind-mounts `/mnt/external-drive` with `propagation: rshared` so that when the backend triggers a host mount via `/api/setup/mount`, the new filesystem becomes visible inside the container. Be careful when changing mount-related code or compose files â€” losing `rshared` breaks the disk-setup flow silently.

**Host Agent (`host-agent/`)** â€” Python asyncio daemon (`agent.py`) installed on the host machine at `/opt/cloudsync-host-agent/`, listening on Unix Domain Socket `/run/cloudsync-host-agent/agent.sock`. Replaces the old named-FIFO bridge (`pipe-daemon.sh`). The backend communicates with it via `HostAgentClient` (Java, `com.cloudsync.client`). Install with `sudo host-agent/install.sh`. Socket is bind-mounted read-only into the backend container at `/run/cloudsync-host-agent`. For WSL2 dev, override `HOST_AGENT_SOCKET` env var.

**Shell scripts at `scripts/host/`** remain as reference/backup. The `scripts/bridge/` directory (pipe-based bridge) is superseded by the host agent and can be removed after migration is validated.

**SQLite location.** The `cloud-sync.db` file lives on the external drive (`EXTERNAL_DRIVE_PATH`, default `/mnt/external-drive` in containers, `./dev-drive` for local non-docker dev). Thumbnails go to `THUMBNAIL_DIR` (default `${EXTERNAL_DRIVE_PATH}/thumbnails`). Flyway migrations under `backend/src/main/resources/db/`.

## Configuration

Key env vars (set in `.env` at repo root for compose, or directly for local runs):

| Variable | Purpose |
|---|---|
| `EXTERNAL_DRIVE_PATH` | Mount point for SQLite + media |
| `APP_USERNAME` / `APP_PASSWORD` | Basic auth for backend API |
| `ICLOUD_SERVICE_URL` | Backend â†’ icloud-service URL (default `http://icloud-service:8000`) |
| `OPENROUTER_API_KEY` | Empty â†’ heuristic fallback agent; set â†’ real LLM |
| `OPENROUTER_MODEL` | Override default model |
| `MICRONAUT_ENVIRONMENTS` | `dev` for local SQLite file, `docker` inside containers |
| `HOST_AGENT_SOCKET` | Path to host agent socket (default `/run/cloudsync-host-agent/agent.sock`) |
